# <a name="microsoft-365-enterprise-documentation"></a>Microsoft 365 Enterprise-dokumentation
Microsoft 365 Enterprise är en komplett, intelligent lösning med Office 365, Windows 10 och Enterprise Mobility + Security (EMS), som ger alla möjlighet att vara kreativa och samarbeta på ett säkert sätt. 

Microsoft 365 Enterprise är utformat för stora organisationer. Här integreras Office 365 Enterprise, Windows 10 Enterprise och EMS. 


## <a name="microsoft-365-services-overview"></a>Översikt över tjänster i Microsoft 365

Det här avsnittet ger en översikt över de EMS- och Office 365-tjänster som ingår i Microsoft 365 Enterprise. Du får också en introduktion av grundläggande begrepp för att du ska förstå hur du bäst använder dem i din organisation. Tjänsterna innehåller funktioner som gör att Microsoft Cloud Enterprise-administratörerna inte bara kan skydda företagets enheter och medarbetarnas identitet utan även styra åtkomsten till företagets data, både vid överföring och lagring.
 
|Tjänst|Beskrivning|
|-------|-----------|
|[Microsoft Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-whatis)|Azure AD innehåller en fullständig uppsättning funktioner för identitetshantering, som multifaktorautentisering, enhetsregistrering, lösenordshantering med självbetjäning, grupphantering med självbetjäning, rollbaserad åtkomstkontroll, övervakning av programanvändning, omfattande granskning samt säkerhetsövervakning och -avisering.|
|[Azure AD Identity Protection](https://docs.microsoft.com/en-us/azure/active-directory/active-directory-identityprotection)|Med den här tjänsten får du hjälp att identifiera potentiella problem som påverkar identiteter i organisationen, och att konfigurera automatiska svar med villkorsbaserade åtkomstprinciper för låg, medelhög och hög inloggningsrisk och användarrisk.|
|[Azure AD Privileged Identity Management](https://docs.microsoft.com/azure/active-directory/active-directory-privileged-identity-management-configure)|Den här tjänsten gör det möjligt för företag och organisationer att minimera antalet personer som har permanent åtkomst till privilegierade åtgärder. I Azure AD Privileged Identity Management introduceras konceptet berättigad administratör. Berättigade administratörer är användare som behöver privilegierad åtkomst då och då, men inte varje dag. Rollen är inaktiv tills användaren behöver åtkomst. Då genomför de en aktiveringsprocess och blir aktiv administratör under en förinställd tidsperiod.|
|[Azure Information Protection](https://docs.microsoft.com/information-protection/understand-explore/what-is-information-protection)| Azure Information Protection är en molnbaserad lösning som ingår i EMS E5-erbjudandet och hjälper företag att klassificera, märka och skydda sina dokument och e-postmeddelanden. Detta kan göras automatiskt av administratörer som definierar regler och villkor, manuellt av användare eller i en kombination där användare får rekommendationer. Du använder Azure Information Protection-etiketter för att klassificera dokument och e-postmeddelanden. När du gör det är klassificeringen alltid identifierbar, oavsett var informationen lagras eller med vem den delas. <br><br>Principinställningarna för Azure Information Protection skyddas av [Azure Rights Management](https://docs.microsoft.com/information-protection/understand-explore/what-is-azure-rms). Det fungerar på liknande sätt som för de etiketter som tillämpas. Skyddet som tillämpas av Rights Management ligger kvar i dokumenten och e-postmeddelandena oavsett var de finns – i eller utanför organisationen, nätverket, filservrarna och programmen.|
|[Microsoft Intune](https://docs.microsoft.com/intune/understand-explore/introduction-to-microsoft-intune)|Intune är en molnbaserad tjänst för hantering av företagsmobilitet (EMM) som hjälper anställda att vara produktiva samtidigt som företagets data skyddas. Intune är tätt integrerat med Azure AD för identitets- och åtkomstkontroll och används för att hantera enheter och program. [Intunes enhetshantering](https://docs.microsoft.com/intune/deploy-use/manage-settings-and-features-on-your-devices-with-microsoft-intune-policies) används för att konfigurera och skydda användarnas enheter, även datorer med Windows. <br><br>Enhetshanteringsfunktionerna i Intune har stöd för både [BYOD](https://docs.microsoft.com/enterprise-mobility-security/solutions/enable-byod)-registrering, där användarna kan registrera sina personliga mobiler, surfplattor och datorer, och [COD](https://docs.microsoft.com/enterprise-mobility-security/solutions/issue-corp-devices)-registrering av företagsägda enheter. Detta möjliggör hanteringsscenarier som automatisk registrering, delade enheter och förgodkända registreringskonfigurationer. För extra säkerhet kan du även kräva multifaktorautentisering när en enhet registreras. När enheten har registrerats för hantering kan Intune konfigurera dess funktioner och inställningar för att aktivera skyddad åtkomst till företagets resurser.|


## <a name="important-concepts-to-understand"></a>Viktiga begrepp
Grundläggande begrepp och EMS-funktioner som du bör känna till beskrivs i tabellen nedan.

|Begrepp|Beskrivning|
|------------|-----------|
|[Azure Multi-Factor Authentication (MFA)](https://docs.microsoft.com/azure/multi-factor-authentication/multi-factor-authentication-get-started-cloud)|Azure MFA är Microsofts lösning för tvåstegsverifiering som hjälper dig att skydda åtkomsten till data och program samtidigt som användarna kan logga in på ett enkelt sätt. Du får stark autentisering via en rad olika verifieringsmetoder, till exempel verifiering via telefonsamtal, textmeddelande eller mobilapp.|
|[Azure AD Conditional Access](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access)|Med den här funktionen i Azure AD kan du använda åtkomstkontroll till molnappar i din miljö utifrån särskilda villkor. Du kan antingen koppla ytterligare åtkomstkrav till kontrollen eller blockera åtkomsten. Den villkorsstyrda åtkomsten baseras på principer.|
|[Exchange Online Skydd mot dataförlust (DLP)](https://support.office.com/article/Overview-of-data-loss-prevention-policies-1966b2a7-d1e2-4d92-ab61-42efbb137f5e)|Principerna i Exchange Online Skydd mot dataförlust (DLP) finns tillgängliga som en premiumfunktion för Exchange Online abonnemang 2 och Office 365-prenumerationer. Med principerna kan företag och organisationer identifiera, övervaka och automatiskt skydda känslig information i hela Office 365.<br><br>Med Exchange Online DLP-principerna kan du identifiera känslig information på flera platser, till exempel i Exchange Online, SharePoint Online och OneDrive för företag. Med principerna kan du till exempel identifiera dokument som innehåller känslig information eller förhindra att känslig information delas av misstag med personer utanför organisationen.|
|[Regler för e-postflöde/transportregler i Exchange](https://support.office.com/en-US/article/Define-mail-flow-rules-to-encrypt-or-decrypt-email-messages-9B7DAF19-D5F2-415B-BC43-A0F5F4A585E8)|Reglerna för e-postflöde i Exchange, som även kallas transportregler, söker efter särskilda villkor eller tillstånd i meddelanden som passerar genom din organisation och agerar på dem. Regler för e-postflöde likar de inkorgsregler som finns tillgängliga i många e-postklienter. Den största skillnaden mellan regler för e-postflöde och de regler som du skapar i ett klientprogram som Outlook är att e-postflödesreglerna tillämpas på meddelanden under överföring, inte när meddelandena har levererats. Regler för e-postflöde innehåller även en bredare uppsättning villkor, undantag och åtgärder. Det ger dig möjlighet att implementera många olika typer av meddelandeprinciper.|
|[Intune Mobile Device Management](https://docs.microsoft.com/intune/understand-explore/introduction-to-microsoft-intune)|Intune möjliggör mobilenhetshantering (MDM) genom att använda de protokoll eller API:er som finns tillgängliga i mobiloperativsystemen. Hanteringen omfattar till exempel att registrera enheter för hantering så att IT-avdelningen har en förteckning över de enheter som har tillgång till företagets tjänster, att konfigurera enheter så att de uppfyller företagets standarder för säkerhets och status, att tillhandahålla certifikat och WiFi-/VPN-profiler för åtkomst till företagets tjänster, att rapportera om och mäta enheternas kompatibilitet med företagets standarder och ta bort företagsdata från hanterade enheter.|
|[Appskyddsprinciper i Intune](https://docs.microsoft.com/intune/deploy-use/protect-app-data-using-mobile-app-management-policies-with-microsoft-intune)|Appskyddsprinciperna i Intune kan användas för att skydda företagets data i mobilappar med eller utan hanteringsregistrerade enheter. Användarnas mobila enheter kan även hanteras av en MDM-lösning som inte är från Microsoft MDM-lösning medan [Intune hjälper dig att skydda information i Office 365](https://docs.microsoft.com/enterprise-mobility-security/solutions/protect-company-data-without-managing-devices). Du kan se till att medarbetarna kan vara produktiva samtidigt som du förhindrar både avsiktlig och oavsiktlig dataförlust. Genom att implementera principer på appnivå kan du begränsa åtkomsten till företagsresurser och hålla data inom IT-avdelningens kontroll.|
|[Livslängd för Azure AD-token](https://docs.microsoft.com/azure/active-directory/active-directory-configurable-token-lifetimes)|Du kan ange livslängd för ett token som utfärdas av Azure Active Directory (Azure AD). Du kan ange token-livslängd för alla program i din organisation, för ett program med flera klientorganisationer eller för en specifik huvudtjänst i organisationen.|
|Microsoft Identity Brokers|Microsoft tillhandahåller program för alla mobilplattformar där man kan använda överbryggning av autentiseringsuppgifter i program från olika leverantörer. Det går att använda specialfunktioner som kräver autentiseringsverifiering från en enda punkt. Sådana program kallas broker-program. I iOS och Android tillhandahålls broker-programmen via Microsoft Authenticator och Intune Company Portal. I Windows 10 tillhandahålls funktionen av Web Authentication Broker, en kontoväljare som är inbyggd i operativsystemet.|
